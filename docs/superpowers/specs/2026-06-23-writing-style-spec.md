# 写作风格功能需求

## 背景

写作风格配置目前只在 Settings 中编辑，且只在选中项目时才可用。用户提出以下改进需求：

1. **写作风格应跟随项目变化**：不同项目有不同的风格需求，当前项目应能直接查看和快速调整风格
2. **编辑器外要能查看当前项目风格**：在日常创作中，需要快速查看当前风格配置
3. **设置中管理所有项目风格**：可以查看、编辑每个项目的风格，并一键复制到当前项目

## 功能需求

### FR-1：工作区风格面板

在工作区界面（大纲/人物/目录/正文编辑器）中，添加一个可折叠的"写作风格"面板或侧边卡片，显示当前项目的风格概要：

- 叙事视角（如：第三人称）
- 正式程度（如：适中）
- 情感强度（如：适中）
- 自定义高频词数量
- 参考文本字数

此面板应：
- 不打断创作流程（可折叠或悬浮）
- 提供快速入口到风格编辑
- 在正文编辑器中尤其重要（风格直接影响写作质量）

### FR-2：Settings 中写作风格管理

在 Settings 的写作风格 tab 中，提供所有项目的风格管理：

- 项目列表，每个项目显示风格概要（叙事视角、正式程度、情感强度）
- 点击项目展开详细编辑
- "一键复制到当前项目"按钮：将另一个项目的风格配置复制到当前项目
- 编辑后保存按钮

### FR-3：风格配置数据模型

每个项目的风格配置包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| narrative_voice | enum | 叙事视角：first_person / third_person / omniscient |
| formality | enum | 正式程度：formal / moderate / casual |
| emotion_intensity | enum | 情感强度：low / moderate / high |
| reference_text | string | 参考文本（最多2000字） |
| custom_stopwords | string(JSON array) | 自定义高频词列表 |

## 优先级

FR-1 > FR-2 > FR-3

FR-1 是最核心的需求（创作时能查看风格），FR-2 是管理需求，FR-3 是底层支撑。
