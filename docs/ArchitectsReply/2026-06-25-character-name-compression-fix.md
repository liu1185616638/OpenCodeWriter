# 人物卡片姓名被压缩问题修复记录

日期：2026-06-25

问题主题：本地测试发现人物卡片列表中角色名称被遮挡或压缩，只显示首字或省略号。

## 问题现象

本地测试截图中，人物列表出现：

- `维克托` 只显示成 `维...`
- `林夏` 只显示成 `林`
- 角色姓名区域被后面的身份/动机摘要挤压

## 根因分析

上一次为了解决“姓名和摘要之间出现异常大空白”的问题，将卡片中间区域改成了：

```tsx
<div className="flex min-w-0 flex-col gap-1 lg:flex-row lg:items-center lg:gap-3">
```

这在大屏下会让“姓名+标签”和“摘要”处于同一行 flex 布局中。

但是摘要字段通常很长，例如：

```text
身份：... | 动机：...
```

当摘要和姓名同处一行时，浏览器会根据 flex 收缩规则重新分配空间。因为姓名部分本身也允许 `min-w-0 truncate`，所以姓名会被长摘要挤压到极窄宽度，最终出现只剩一个字或省略号的情况。

## 修复方案

修改文件：

- `src/views/CharacterEditor.tsx`

### 1. 姓名和摘要拆成两行稳定结构

不再使用大屏 `lg:flex-row` 把姓名和摘要强行放到同一行，而是改为：

```tsx
<div className="grid min-w-0 gap-1">
```

这样：

- 第一行只负责显示姓名和角色层级标签。
- 第二行单独显示身份/动机摘要。
- 摘要不再参与挤压姓名。

### 2. 姓名设置合理最大宽度并禁止被摘要压缩

姓名 span 调整为：

```tsx
<span className="max-w-40 shrink-0 truncate font-semibold text-foreground sm:max-w-56 lg:max-w-72" title={character.name}>
```

关键点：

- `shrink-0`：姓名不会被摘要压缩到只剩一个字。
- `max-w-40 / sm:max-w-56 / lg:max-w-72`：长姓名仍会在合理宽度内截断，避免反过来撑爆卡片。
- `title={character.name}`：如果姓名被截断，鼠标悬停仍能看到完整名称。

### 3. 摘要独立截断

摘要调整为：

```tsx
<span className="block min-w-0 truncate text-sm text-muted-foreground" title={subtitle}>
```

这样摘要只在自己的行内截断，不再影响姓名显示。

## 预期效果

- `维克托`、`林夏` 等短姓名应完整显示。
- 很长的人物名会在合理最大宽度内截断，而不会遮挡标签和箭头。
- 身份/动机摘要仍然保留，但作为第二行文本单独截断。
- 卡片整体仍然保持自适应宽度，不会恢复之前的大空白问题。
