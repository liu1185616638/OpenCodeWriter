# 制作设计稿

## 输入

读取 `Product-Spec.md`、`Design-Brief.md` 和现有设计文件。确认设计工具、目标文件和可复用设计系统可访问。

## 原则

- 先发现并复用现有组件、变量、样式和令牌，再创建新组件。
- 先建立可复用基础组件，再组装页面；相同元素保持单一来源。
- 为 Spec 中每个有 UI 的功能创建对应页面，不遗漏失败和恢复路径。
- 使用接近真实长度和语义的内容，不用无意义占位文本。
- 修改 UI 时同步更新设计稿；代码不能成为未记录的设计分支。

## 产出

在连接的设计工具中生成组件、页面、状态变体和必要原型连接，并在回复中列出文件、页面及覆盖的 Spec 条目。

## 验收

- 页面清单与 Design Brief 一一对应。
- 每个页面覆盖默认、空、加载、错误和关键成功状态。
- 颜色、字体、间距和圆角来自变量或统一令牌。
- 组件实例可复用，真实内容不会溢出关键布局。

## 停止条件

缺少设计 MCP、目标文件权限、关键需求或设计方向时停止并明确缺项；不要用文字说明冒充已创建的设计稿。
