# 演进规则

## 输入

读取原始信号、当前 `AGENTS.md`、相关 Skill、`.codex/EVOLUTION.md` 和既有建议。保留用户原话，不把一次性偏好自动提升为通用规则。

## 原则

- 合并指向同一根因的信号，区分通用失败模式与项目专属偏好。
- 每条建议说明触发证据、目标文件、精确改动、预防的失败和可能副作用。
- 同时扫描重复、失效、已内化或从未触发的规则，主动提出退休。
- 规则总量可以减少；更精炼、更准确优先于更全面。
- 只提议，不替用户决定，不在后台修改任何规则。

## 产出

更新 `.codex/evolution/proposals.md`，按独立条目列出新增、修改、退休或丢弃建议，并逐条请求用户确认。

## 验收

- 每条建议可追溯到信号或明确的规则冲突。
- 通用规则和项目记忆不混写。
- 用户批准后才修改目标文件；拒绝后删除对应建议和已消费信号。
- 修改后检查重复和矛盾，保持净增量最小。

## 停止条件

信号含义不清、建议会影响多个不同范围或无法判断应进入规则还是项目偏好时，向用户确认。
