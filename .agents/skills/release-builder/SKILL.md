# 构建发布

## 输入

读取发布目标、平台、版本策略、构建命令、测试要求和回滚方式。发布前读取 [references/privacy-audit.md](references/privacy-audit.md)。

## 原则

- 从干净状态构建最终产物，不复用无法证明来源的旧产物。
- 从安装包或部署包测试，不从源码目录或开发服务器代替验证。
- 分别检查路径、依赖、权限、配置和升级/卸载行为。
- 隐私审计是绝对门禁；命中敏感项立即停止，修复后重新构建并从头审计。
- 未经用户明确授权，不执行真实发布、上传、签名、推送标签或修改线上环境。

## 产出

生成发布产物、校验值和发布报告，记录构建命令、产物路径、安装/启动验证、测试结果、隐私审计和已知限制。

## 验收

- 在目标环境从最终产物完成安装、启动、核心流程和卸载/回滚验证。
- 产物不包含开发者绝对路径、环境文件、数据库、密钥或明文凭据。
- 版本、变更记录、许可证和校验值一致。
- 所有声明附刚运行的证据；仅“构建成功”不能判定可发布。

## 停止条件

发现敏感内容、签名或权限缺失、目标环境不可用、测试失败或需要外部发布授权时停止。
