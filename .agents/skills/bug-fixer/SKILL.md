# 修复缺陷

## 输入

收集复现步骤、期望结果、实际结果、日志、失败测试、环境和最近相关变更。没有稳定复现时，先把问题转成最小失败测试或可观察探针。

## 原则

- 不猜不试；没有证据不下结论。
- 同时最多保留三个假设，按可能性排序，一次只验证一个。
- 记录被否定假设及证据，不重复验证。
- 一次只改一个原因相关位置；需要多文件原子修改时说明共同原因。
- 先确认根因，再写失败测试，再做最小修复。
- 连续三次修复未解决同一问题时强制停下，重审需求、环境和系统边界。

## 产出

提供复现测试、根因证据、最小补丁、回归测试以及修复前后验证输出。

## 验收

- 失败测试在修复前按预期失败，修复后通过。
- 根因解释能连接输入、代码路径和观察结果。
- 相关回归测试、类型检查和构建通过。
- 没有靠放宽断言、吞异常或改变期望来制造绿色结果。

## 停止条件

无法复现、证据互相矛盾、需要产品决策或连续三次失败时停止，列出已排除项和下一项最小决策。
