# 构建功能

## 输入

开工前重读相关 Spec 条目、设计稿、Design Brief、DEV-PLAN 原文和涉及代码。先搜索影响范围、引用点和可复用组件。

## 原则

- 只修改当前任务直接需要的代码；匹配现有风格，不顺手重构。
- 优先复用已有组件、样式和工具，不为一次性需求增加抽象。
- 先写能重现目标行为的测试并确认失败，再写最小实现使其通过。
- 按真实数据和真实行为实现按钮、数字、卡片和引导；不留装饰性假功能。
- 删除本次修改产生的死代码，不清理预先存在的无关问题。
- 变更需求时先改文档并获得确认，再动代码。

## 产出

交付最小代码和测试变更，更新 DEV-PLAN 对应清单。完成声明同时给出刚运行的命令、关键输出和涉及文件。

## 验收

- 新测试曾因缺少目标行为而失败，现已通过。
- 相关测试、类型检查、编译和功能验证当场通过。
- 实现逐条满足 Spec，视觉值来自设计源而非记忆。
- 没有越过当前 Phase、没有硬编码演示数据、没有未说明占位。

## 停止条件

需求冲突、需要新增生产依赖、会触及计划外架构或同一问题连续三次失败时停止并请求决策。
