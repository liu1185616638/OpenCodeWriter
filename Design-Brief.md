# Design Brief — OpenCodeWriter

## 文档定位与权威顺序

本文件描述 OpenCodeWriter 当前 Tauri 桌面应用的产品形态、信息架构、交互原则和设计验收基线。

视觉与交互冲突时按以下顺序判断：已确认设计稿 > 本 Design Brief > Product Spec 中的视觉描述。功能逻辑和验收标准仍以 `Product-Spec.md` 为准；技术实现边界以 `AGENTS.md` 和 `DEV-PLAN.md` 为准。

## 产品形态

OpenCodeWriter 是基于 Tauri 2、React 19、TypeScript 与 SQLite 的单窗口桌面写作工作台，服务长期小说创作。产品采用可调整的多栏工作区，而不是终端 TUI：

- 桌面端优先，支持鼠标、触控板和完整键盘操作。
- 默认三栏：全局导航、任务主区、上下文检查器；在专注写作时可收起两侧栏。
- 以本地项目和文档为中心，AI 是嵌入创作流程的协作者，不使用聊天窗口统领整个产品。
- SDK-backed Runtime 是默认 AI 底座；普通设置不暴露 Runtime 切换，内部 fallback 不作为用户心智。
- 不参考或复用当前前端页面，本规范从已确认需求、当前后端能力和开发计划重建设计基线。

## 核心用户结果

1. 用户能从一句想法建立项目，沿“大纲 → 人物与世界 → 章节规划 → 正文”持续推进。
2. 当前任务、上游依据、下游影响和 AI 运行状态始终可解释。
3. 长篇写作中的角色状态、事实、伏笔、知识资料和写法规则可被积累、检索和审阅。
4. AI 失败、修复或覆盖内容时不丢稿，并能通过快照和任务记录恢复。
5. 新手得到明确下一步，专业作者可以直接进入高密度编辑与批量工作。

## 设计原则

### 创作为主，AI 为辅

正文和结构化创作内容占据最大空间。AI 操作靠近当前编辑对象，但不长期占用大块聊天区；思考过程、工具调用和运行日志进入可折叠的任务抽屉。

### 上下文可见

每次生成前显示将使用的关键上下文摘要，包括大纲、出场人物、章节任务单、写法规则和知识来源。用户不需要手动拼接上下文，但可以确认系统依据。

### 状态可解释

“未开始、可开始、进行中、已完成、过时、生成中、失败”同时使用颜色、图标和文字。过时状态必须展示来源与影响，不能只有警告色。

### 渐进披露

主界面只保留当前任务和一到两个支持区域。详细资产、日志、模型路由、权限和恢复操作进入检查器、抽屉或独立页面，避免把所有后端能力堆成仪表盘卡片。

### 可逆优先

覆盖大纲、章节或正文前显示快照提示；修复结果先预览差异，再由用户确认写入。删除项目、批量覆盖等不可逆操作必须二次确认。

## 应用外壳

### 左侧全局导航

- 展开宽度 248px，收起宽度 56px。
- 顶部为项目切换器；中部按“创作、资产、支持”分组；底部为任务中心、设置和连接状态。
- 当前项目与当前页面有清晰选中态；过时数量、失败任务数量用克制的计数标记展示。
- 阶段顺序始终可见，但不把每个状态都做成独立卡片。

### 顶部任务栏

- 高度 56px。
- 左侧显示页面标题、当前对象和保存状态；右侧显示模型选择、上下文入口和当前主操作。
- 全局搜索/命令面板使用 `Ctrl/Cmd+K` 打开，不在顶栏常驻大型搜索框。

### 主工作区

- 承载编辑器、列表或画布，是视觉焦点。
- 长文编辑最大阅读宽度 760px；结构化规划可以使用完整可用宽度。
- 主要操作固定在标题区或底部生成栏，不能同时出现多个同权重主按钮。

### 右侧检查器

- 默认宽度 320px，可在 288–420px 内拖动，可完全收起。
- 按当前对象显示任务单、人物、上下文、审核问题或属性；不作为永久的功能目录。
- 同一时间只展开一个高优先级面板，其他内容折叠。

### 底部任务抽屉

- 默认收起为 32px 状态条；运行 AI 或批量任务时可展开至窗口高度的 35%。
- 区分正文输出、thinking、tool、skill、mcp、error 和 done 事件。
- 允许取消任务、查看错误、重试和定位生成目标。

## 页面与关键状态

### V1 首次配置

三步向导：连接方式 → 模型确认 → 完成并创建项目。

| 状态 | 设计要求 |
|---|---|
| 默认 | 单列 560px 表单，每步只强调一个决定 |
| 空 | 提供 API 地址示例和隐私说明，Key 默认隐藏 |
| 加载 | 连接测试按钮显示进度，输入保持可见但不可重复提交 |
| 错误 | 就地说明连接失败原因，保留全部输入，提供重试 |
| 成功 | 明确显示已连接的模型名称，主按钮进入开书 |

### V2 项目库与一句话开书

项目库展示最近项目的名称、类型、字数、当前阶段和最后编辑时间。创建流程支持直接新建，也支持输入一句故事想法后生成三个结构化方向。

| 状态 | 设计要求 |
|---|---|
| 空 | 解释“从一句想法开始”，主按钮为“创建第一本书” |
| 默认 | 最近项目使用列表而非卡片墙，首行可继续上次创作 |
| 生成方向 | 三个方向并排比较题材、主角、冲突和卖点，只允许选择一个继续 |
| 模型缺失 | 不创建半成品项目，直接引导配置模型 |
| 重名 | 输入框就地报错并保留内容 |
| 删除 | 对话框展示项目名和影响范围，危险按钮与取消按钮分离 |

### V3 项目定盘

结构化编辑书名、类型、受众、基调、核心卖点、目标篇幅和创作约束。用于开书后的书级 framing，也是大纲生成的重要上下文。

| 状态 | 设计要求 |
|---|---|
| 空 | 按重要性分组字段，给出简短例子，不显示大面积空白 |
| 编辑 | 自动保存，并在标题区显示“已保存/保存中/保存失败” |
| 完成 | 显示进入大纲的明确下一步 |
| 修改影响 | 若已有大纲，保存前说明可能导致哪些内容过时 |

### V4 大纲工作台

主区为纯文本/Markdown 风格大纲编辑器；检查器显示项目定盘、结构提示和上下文清单。

| 状态 | 设计要求 |
|---|---|
| 空 | 展示可编辑模板骨架，主操作为“生成大纲” |
| 默认 | 章节结构和故事正文使用不同层级，编辑器保持长文阅读宽度 |
| 生成中 | 内容流式写入预览区，旧内容保持可恢复；可取消 |
| 失败 | 保留已有大纲，错误条提供重试和切换模型 |
| 成功 | 新内容先进入可审阅状态，再保存并标记完成 |
| 下游影响 | 修改后展示人物、章节和正文的过时原因与处理入口 |

### V5 人物与关系

人物页采用“分层列表 + 人物详情”，支持主要角色、重要配角和其他角色。关系视图是辅助页，不替代人物编辑。

| 状态 | 设计要求 |
|---|---|
| 空 | 上游未完成时说明原因；可开始时突出“基于大纲生成人物” |
| 默认 | 列表显示姓名、身份、层级和最近状态；详情编辑身份、外貌、性格、动机、关系和关键事件 |
| 关系 | 以可读关系列表为主，必要时再使用轻量关系图；每条关系可编辑类型 |
| 状态演变 | 按章节时间线查看目标、情绪、地点和状态摘要 |
| 过时 | 顶部说明大纲何处变化，提供重新生成或标记最新 |
| 删除 | 若人物被章节或关系引用，明确展示引用影响 |

### V6 世界观

左侧按地点、组织、规则、物件等类型筛选；主区编辑名称、描述、规则和关联内容。

| 状态 | 设计要求 |
|---|---|
| 空 | 提供常用世界条目类型和新增入口 |
| 默认 | 使用分组列表与详情，不使用无边界卡片瀑布流 |
| 搜索 | 结果突出命中字段并保留类型上下文 |
| 删除 | 被故事事实或章节引用时提示影响 |

### V7 章节规划

主区为可排序章节列表，选中章节后显示任务单。任务单覆盖 POV、目标、冲突、转折、结果、出场人物、场景和预计字数。

| 状态 | 设计要求 |
|---|---|
| 空 | 说明需要大纲和人物，条件满足后突出生成入口 |
| 默认 | 行内显示章号、标题、摘要、状态和正文进度，列表支持拖动排序 |
| 选中 | 右侧检查器编辑完整任务单并显示上下游引用 |
| 生成中 | 批量生成显示总进度和已完成数量，不阻塞查看已完成章节 |
| 过时 | 标注具体受到大纲、人物或前序章节哪项修改影响 |
| 失败 | 已生成章节保留，失败章节可单独重试 |

### V8 正文工作室

默认三栏：章节列表、正文编辑器、章节检查器。专注模式收起两侧栏，只保留编辑器和最小状态。

| 状态 | 设计要求 |
|---|---|
| 空 | 选中章节后说明将使用的任务单和上下文，主操作为“生成本章” |
| 默认 | 正文采用舒适行宽与行高；标题区显示字数、保存状态和章节状态 |
| 生成中 | 流式内容与用户正文明确分区，确认前不覆盖旧内容 |
| 高频词 | 用下划线与侧边问题列表双重呈现，支持逐项忽略或替换 |
| 过时 | 提供原因、影响范围、重新生成和保留现稿四类信息 |
| 保存失败 | 保留本地编辑状态，持续提示直到恢复或用户处理 |

### V9 章节审核、修复与后护理

审核结果在正文右侧检查器展示总分、连续性、人物一致性、节奏和问题列表。修复结果使用并排或行内差异预览；后护理提取的事实、伏笔和人物状态必须先审阅再写入。

| 状态 | 设计要求 |
|---|---|
| 未审核 | 说明审核依据和预计输出 |
| 审核中 | 保持正文可读，任务抽屉显示运行阶段 |
| 有问题 | 先列高严重度问题，点击可定位正文位置 |
| 修复预览 | 明确原文与建议差异，主操作为“应用修复” |
| 修复失败 | 不修改正文，保留审核结果与错误详情 |
| 后护理 | 按事实、伏笔、人物状态分组确认，可逐项排除 |

### V10 故事事实与伏笔

事实页按类型和章节筛选；伏笔页按“待回收、已回收、已放弃”分组，并显示埋设与回收章节。

| 状态 | 设计要求 |
|---|---|
| 空 | 从章节后护理导入或手动新建，两种入口权重明确 |
| 默认 | 使用可扫描列表，置信度是辅助信息而非核心指标 |
| 待回收 | 显示已跨越章节数和建议回收位置 |
| 已回收 | 同时显示 setup 与 payoff 章节 |

### V11 知识库

支持导入文本资料、查看来源和受限数量的搜索结果。用户能够确认哪些资料将进入当前 AI 上下文。

| 状态 | 设计要求 |
|---|---|
| 空 | 解释本地知识库用途和支持的导入方式 |
| 导入中 | 显示来源名称和处理状态 |
| 搜索 | 结果包含标题、片段和来源，不无限加载 |
| 注入确认 | 生成前列出被选中的知识片段，可取消单项 |
| 错误 | 不删除已导入来源，提供重新处理或移除 |

### V12 写法引擎与风格

风格页包含参考文本、叙事视角、正式度、情感浓度和自定义高频词；规则页管理可启用/禁用的写法规则。

| 状态 | 设计要求 |
|---|---|
| 默认 | 参考文本显示 2000 字计数；参数使用有解释的选择控件 |
| 超限 | 在输入处即时提示截断影响，不等保存后才报错 |
| 规则空 | 提供从文本提取规则和手动新增入口 |
| 提取中 | 新规则先进入待审阅区，不直接启用 |
| 已禁用 | 保留规则内容和来源，但视觉降级 |

### V13 任务中心、生成记录与快照

任务中心面向可恢复的生成和批量任务；生成记录用于排障；快照用于内容恢复，三者分栏或分标签展示，避免混成日志流。

| 状态 | 设计要求 |
|---|---|
| 运行中 | 显示目标、阶段、模型、进度、开始时间和取消操作 |
| 失败 | 展示可读错误、已完成结果和重试范围 |
| 已完成 | 可跳转到生成目标，不把成功记录长期高亮 |
| 快照 | 显示原因、时间、目标和内容摘要；恢复前提供预览 |
| 空 | 分别解释任务、日志和快照的用途 |

### V14 设置

| 分区 | 设计要求 |
|---|---|
| 模型预设 | 新增、编辑、测试和删除预设；API Key 默认隐藏 |
| 模型路由 | 按大纲、人物、章节、正文、审核、修复、润色等任务选择主模型和 fallback 模型 |
| Runtime | 不向普通用户暴露 SDK-backed/OpenAI-compatible 切换 |
| 工具与权限 | 展示业务工具权限和最近审批；危险写入默认拒绝 |
| MCP | 实验性、默认关闭；当前执行链未完整可用时明确说明，不伪装成可用连接 |
| 外观 | 明暗主题、界面密度和编辑器字号；不提供任意字体上传 |
| 关于 | 版本、数据目录和资源状态 |

## 全局状态矩阵

### 创作阶段

| 状态 | 表现 | 操作 |
|---|---|---|
| 未开始 | 中性空心图标 + “未开始” | 上游未完成时只读 |
| 可开始 | 强调色图标 + “可开始” | 进入、生成或手动创建 |
| 进行中 | 实心图标 + 保存状态 | 编辑、生成、标记完成 |
| 已完成 | 成功色对勾 + “已完成” | 编辑、重新生成 |
| 过时 | 警告图标 + 原因数量 | 查看原因、重新生成、标记最新 |

### AI 任务

| 状态 | 表现 | 数据保护 |
|---|---|---|
| 准备 | 上下文摘要、模型和预计影响 | 不修改目标 |
| 运行 | 进度、流式输出、当前阶段、取消 | 旧内容保持可恢复 |
| 待审阅 | 预览或差异视图 | 未确认不覆盖 |
| 成功 | 简短成功反馈并可跳转 | 保存生成日志 |
| 失败 | 可读错误、重试和切换模型 | 保留旧内容与已完成子任务 |

### 保存与恢复

| 状态 | 表现 |
|---|---|
| 已保存 | 标题区低强调文字 |
| 保存中 | 小型进度指示，不阻塞输入 |
| 保存失败 | 持续错误提示 + 重试，不能伪装成已保存 |
| 已创建快照 | 在覆盖确认中显示快照时间 |
| 可恢复 | 预览快照后确认恢复，恢复操作本身再创建快照 |

## 视觉方向基线

最终视觉方向由后续视觉选案确定，但所有方向必须满足以下硬约束：

- 桌面生产力工具，而非营销网站、聊天应用或卡片仪表盘。
- 明确区分“创作内容、结构化资产、AI 运行信息”三种信息层级。
- 暗色与亮色均可用；默认主题不得使用纯黑背景或高饱和霓虹大面积铺色。
- 正文区域优先保证中文长文阅读；系统 UI 优先保证紧凑扫描。
- 依靠间距、对齐和字重建立层级，边框次之，阴影只用于浮层。

## 设计令牌

视觉选案前先锁定语义和尺寸，具体色值可在选案后调整。

### 排版

| 用途 | 规格 |
|---|---|
| UI 字体 | 中文系统无衬线字体栈，正文基准 14px |
| 编辑器正文 | 16px，行高 1.75，最大阅读宽度 760px |
| 数据与代码 | 等宽字体，12–13px，仅用于日志、快捷键和技术值 |
| 页面标题 | 20px / 600 |
| 区域标题 | 14px / 600 |
| 辅助文字 | 12px / 400 |

### 间距与尺寸

| 令牌 | 值 |
|---|---|
| 基础单位 | 4px |
| 紧凑间距 | 4 / 8px |
| 常规间距 | 12 / 16px |
| 区域间距 | 24 / 32px |
| 控件高度 | 32px 紧凑、36px 常规、40px 主要 |
| 顶部任务栏 | 56px |
| 圆角 | 6px 控件、8px 面板、12px 浮层 |
| 分隔线 | 1px 语义边界色 |

### 语义颜色

每个主题都必须提供：`canvas`、`surface`、`surface-raised`、`text-primary`、`text-secondary`、`text-muted`、`border`、`accent`、`accent-soft`、`success`、`warning`、`danger`、`info`、`selection` 和 `focus-ring`。状态色不得作为大面积背景使用。

## 组件体系

设计稿优先建立可复用组件，再组装页面：

- 应用外壳：项目切换器、导航分组、导航项、顶栏、检查器、任务抽屉。
- 操作：主按钮、次按钮、图标按钮、危险按钮、分段控件、下拉选择、命令项。
- 输入：单行输入、多行输入、选择器、开关、滑块、标签输入、搜索框。
- 内容：编辑器、章节行、人物行、属性组、问题行、差异块、上下文来源行。
- 状态：阶段标记、保存状态、过时条、错误条、空状态、加载骨架、进度行。
- 浮层：确认对话框、命令面板、模型选择、上下文预览、恢复预览。

所有组件至少覆盖默认、悬停、聚焦、按下、禁用和错误状态；可编辑列表还需覆盖空、加载和操作中状态。

## 交互与动效

- 高频操作可通过鼠标和键盘完成；焦点样式始终可见。
- 页面切换不使用大幅位移动画。面板展开、状态变化和浮层使用 120–180ms 的淡入或小幅移动。
- AI 流式输出不自动抢夺滚动位置；仅当用户位于内容末尾时跟随。
- 自动保存采用 500–800ms 防抖，并持续展示真实保存状态。
- 拖动仅用于章节排序和可调整栏宽；所有拖动操作提供键盘替代。

## 窗口与响应式行为

- 推荐设计画板：1440 × 960；同时检查 1280 × 800。
- 最小可用窗口：1100 × 720。低于该尺寸时优先收起右侧检查器，其次收起左侧导航文字。
- 左侧导航和右侧检查器可收起；主编辑区不得低于 640px。
- 对话框最大宽度 640px，超长设置使用独立页面而非嵌套对话框。
- 不支持移动端；窄窗口适配只服务桌面分屏和小尺寸笔记本。

## 无障碍

- 正文与 UI 文本满足 WCAG AA 对比度；焦点环与背景对比清晰。
- 状态使用图标、文字和颜色三重编码。
- 表单错误与字段关联，屏幕阅读器可以读取。
- 图标按钮必须有文字标签或可访问名称。
- 支持 200% UI 缩放，主流程不丢失操作入口。
- 减少动态效果设置应关闭非必要动画。

## 明确禁用项

- 不设计成以聊天气泡为中心的 AI 应用。
- 不使用卡片套卡片、指标大屏或无意义的首页统计。
- 不在普通设置中暴露 Runtime 实现切换。
- 不把 thinking 内容混入作品正文。
- 不在未确认前自动覆盖正文、大纲或修复结果。
- 不把未完整落地的 MCP 执行能力表现为可用功能。
- 不使用纯装饰渐变、玻璃拟态或高饱和霓虹干扰长时间写作。
- 不依赖悬停才能发现关键操作。
- 不做移动端页面、云同步、多人协作、ePub/PDF 导出或 Skill 可视化编辑器。

## 设计稿验收

- 页面清单覆盖 V1–V14；优先完整绘制 V2、V4、V5、V7、V8、V9、V12、V14。
- 每个核心页面至少有默认态，并以独立状态板覆盖空、加载、错误、关键成功和过时/恢复路径。
- 组件、色彩、字体、间距和圆角来自统一变量或组件实例。
- 真实中文内容在 1280 × 800 与 1440 × 960 画板中不溢出关键布局。
- 正文生成、审核、修复、后护理、快照恢复和 MCP 禁用边界均不误导用户。
