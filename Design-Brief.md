# Design Brief — OpenCodeWriter

## 产品形态

终端桌面应用（TUI），基于 OpenTUI + TypeScript + bun 构建。单窗口全屏运行，键盘优先操作，鼠标辅助选择。IDE 式分屏布局，信息密度高，适合长时间深度创作。

## 设计目标

1. **任务连贯**：大纲→人物→目录→正文的递进流程始终可见，当前阶段、上游状态、下游进度一目了然
2. **信息密度**：紧凑布局最大化单屏信息量，专业用户减少滚动和切换
3. **状态透明**：过时标记、生成进度、模型状态等关键信息始终可见，不需主动查询
4. **操作高效**：键盘快捷键覆盖所有高频操作，AI 生成/编辑/切换阶段无需离开键盘
5. **双主题**：暗色+亮色可切换，适应不同使用环境

## 信息架构

```
根
├─ 配置向导（首次启动）
├─ 项目列表
│   ├─ 创建项目
│   └─ 删除项目（确认对话框）
└─ 项目工作区
    ├─ 左侧面板
    │   ├─ 项目列表区
    │   ├─ 创作阶段导航
    │   │   ├─ 大纲 ✓/●/○
    │   │   ├─ 人物 ✓/●/○
    │   │   ├─ 章节目录 ✓/●/○
    │   │   └─ 正文 ✓/●/○
    │   ├─ Skills 列表
    │   └─ 底部状态栏（当前模型/连接状态）
    └─ 右侧面板
        ├─ 编辑区（大纲/人物/目录/正文）
        ├─ 状态提示区（过时标记/高频词标记/AI 反馈）
        └─ 操作栏（AI 生成/模型选择/保存）
```

## 页面/视图清单

### V1 配置向导

首次启动时全屏展示，引导用户配置 AI 模型。

| 状态 | 表现 |
|------|------|
| 默认 | 三步表单：① API 地址 ② API Key ③ 模型名称，每步聚焦一个输入 |
| 空 | 各字段显示 placeholder 示例（如 `https://api.openai.com/v1`） |
| 错误 | API 连接测试失败时，红色提示 + "重试"按钮 |
| 成功 | 显示"连接成功"，进入项目列表 |

### V2 项目列表

展示所有项目，创建/切换/删除。

| 状态 | 表现 |
|------|------|
| 默认 | 紧凑列表，每行显示：项目名 + 进度指示（大纲✓ 人物● 目录○ 正文○） |
| 空 | 提示"创建第一个项目开始创作"，聚焦"新建"按钮 |
| 创建中 | 底部弹出输入框输入项目名，Enter 确认，Esc 取消 |
| 删除确认 | 模态确认框："确定删除 [项目名]？此操作不可撤销" [确认/取消] |
| 重名错误 | 输入框下方红色提示"项目名已存在" |

### V3 项目工作区

核心视图，分屏布局。

#### 左侧面板（宽度约 25%）

| 区域 | 内容 |
|------|------|
| 项目列表 | 当前项目高亮，其他项目单行显示，点击切换 |
| 创作阶段 | 四个阶段节点，图标表示状态：✓ 已完成、● 当前/进行中、○ 未开始、⚠ 过时 |
| Skills 列表 | 当前阶段可用 Skill，点击执行 |
| 底部状态栏 | 当前模型预设名 + API 连接状态指示灯（●绿/●红） |

#### 右侧面板（宽度约 75%）

根据左侧选中的阶段切换内容：

**V3a 大纲编辑**

| 状态 | 表现 |
|------|------|
| 默认 | 全尺寸文本编辑区，底部操作栏：[AI生成] [模型▼] [保存] |
| 空 | 编辑区显示大纲模板结构作为参考（灰色淡显） |
| AI 生成中 | 编辑区显示生成进度条，底部状态"正在生成大纲..."，操作栏禁用 |
| AI 失败 | 编辑区保留已有内容，底部红色提示错误信息 + [重试] |
| 高频词标记 | AI 味高频词以特殊颜色下划线标记，hover 显示建议替换词 |

**V3b 人物编辑**

| 状态 | 表现 |
|------|------|
| 默认 | 三层级折叠列表：▼ 主要角色 → ▼ 重要配角 → ▼ 其他角色，点击展开角色卡片 |
| 空 | 提示"基于大纲生成人物"，[AI生成] 按钮高亮 |
| AI 生成中 | 底部进度条 + "正在生成人物小传..." |
| 上游过时 | 顶部黄色提示条："大纲已修改，人物可能需要更新 [重新生成] [标记为最新]" |
| 角色编辑 | 展开角色卡片，显示姓名/身份/外貌/性格/动机/关系/事件字段，可逐项编辑 |

**V3c 章节目录**

| 状态 | 表现 |
|------|------|
| 默认 | 列表显示：序号 + 标题 + 摘要（单行预览），可拖拽排序 |
| 空 | 提示"基于大纲和人物生成章节目录"，[AI生成] 按钮高亮 |
| AI 生成中 | 底部进度条 + "正在生成章节目录..." |
| 上游过时 | 顶部黄色提示条："大纲/人物已修改，目录可能需要更新 [重新生成] [标记为最新]" |
| 选中章节 | 高亮当前章节，右侧显示摘要全文编辑区 |

**V3d 正文编辑**

| 状态 | 表现 |
|------|------|
| 默认 | 左侧章节列表（仅标题），右侧当前章节正文编辑区 |
| 空 | 选中章节后，编辑区提示"生成该章正文"，[AI生成] + [模型▼] |
| AI 生成中 | 编辑区显示生成进度，已生成部分实时流式显示 |
| 上游过时 | 顶部黄色提示条："上游内容已修改，正文可能需要更新 [重新生成] [标记为最新]" |
| 高频词标记 | AI 味高频词以特殊颜色下划线标记，hover 显示建议替换词 |
| Skill 可用 | 底部显示当前可用 Skill（如润色），点击执行 |

### V4 设置视图

从左侧面板底部入口进入，覆盖右侧面板。

| 区域 | 内容 |
|------|------|
| 模型预设 | 列表显示已配置预设，可新增/编辑/删除，每项包含：名称、API地址、Key、模型名 |
| 写作风格 | 风格参考文本编辑区（2000字上限，右下角字数统计）、拟人化参数下拉、高频词表编辑 |
| 主题切换 | 暗/亮主题切换 |
| 关于 | 版本号、资源目录路径 |

## 状态矩阵

### 创作阶段状态

| 状态 | 图标 | 含义 | 可操作 |
|------|------|------|--------|
| 未开始 | ○ | 该阶段无内容，且上游未完成 | 仅查看（若上游未完成则不可点击） |
| 可开始 | ◉ | 上游已完成，可进入该阶段 | 点击进入、AI 生成 |
| 进行中 | ● | 有内容但未标记完成 | 编辑、AI 生成、手动编辑 |
| 已完成 | ✓ | 用户确认内容满意 | 编辑、重新生成 |
| 过时 | ⚠ | 上游已修改，当前内容可能不一致 | 重新生成、标记为最新 |

### 状态流转

```
○ ─(上游完成)─→ ◉ ─(首次生成/编辑)─→ ● ─(用户确认)─→ ✓
                                        ↑                  │
                                        └──(继续编辑)───────┘
                                                              │
                                        ┌──(上游修改)─────────┘
                                        ↓
                                        ⚠ ─(重新生成)─→ ●
                                        ⚠ ─(标记为最新)─→ ✓
```

### AI 生成状态

| 状态 | 表现 | 用户可操作 |
|------|------|-----------|
| 空闲 | 操作栏按钮可点击 | 触发生成 |
| 生成中 | 进度条 + 流式输出 + 操作栏禁用 | 取消（Esc） |
| 成功 | 内容填充到编辑区 | 编辑、保存、重新生成 |
| 失败 | 红色错误提示 + 保留原有内容 | 重试、切换模型重试 |

### 过时标记状态

| 修改源 | 受影响下游 | 标记行为 |
|--------|-----------|---------|
| 大纲修改 | 人物 ⚠ + 目录 ⚠ + 正文 ⚠ | 全部标记 |
| 人物修改 | 目录 ⚠ + 正文 ⚠ | 不影响大纲 |
| 目录修改 | 正文 ⚠ | 不影响大纲和人物 |
| 正文修改 | 无 | 正文自身修改不触发过时 |

## 组件原则

1. **文本编辑器**：支持基础编辑（光标移动/选择/复制/粘贴/撤销），不做富文本，内容纯文本存储
2. **列表组件**：紧凑行高，支持键盘导航（↑↓ 选择，Enter 展开/确认，Space 标记），拖拽排序仅用于章节目录
3. **按钮**：文字按钮，[方括号]样式，键盘可通过 Tab/快捷键聚焦
4. **输入框**：单行底部弹出式（项目名创建等），多行内嵌式（编辑区），focus 高亮边框
5. **模态框**：最小化使用，仅用于删除确认等不可逆操作
6. **提示条**：顶部黄色条（过时警告）、底部红色条（错误），单行，可关闭
7. **进度条**：底部状态区，显示 AI 生成进度，流式输出时在编辑区实时显示

## 窗口行为

- 全屏终端运行，不响应窗口缩放（TUI 固定布局）
- 最小终端尺寸：120 列 × 30 行，低于此尺寸显示提示并拒绝启动
- 左右面板比例固定 25%:75%，不响应拖拽调整

## 无障碍

- 所有操作可通过键盘完成，不依赖鼠标
- 状态变化通过颜色+符号双重编码（不仅依赖颜色）
- 焦点位置始终可见，focus 样式明确
- 错误提示同时使用颜色和文字描述

## 明确禁用项

- 不做动画/过渡效果（TUI 性能和兼容性考量）
- 不做拖拽调整面板宽度
- 不做富文本编辑
- 不做自定义字体选择
- 不做嵌套模态框
- 不做内联通知堆叠（同一时间只显示一条提示条）
